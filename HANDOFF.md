# HANDOFF — развёртывание копии системы под другой клуб

Это руководство, как поднять полную копию сайта + системы турнир-директора
(как на `zacep.fun`) для другого покерного клуба. Стек: **статический фронт
(HTML/CSS/JS) + PHP 8 + MySQL**. Никаких фреймворков и сборки.

---

## 1. Что входит в систему

**Публичный сайт**
- `index.html` — лендинг (hero, турниры из БД, «как проходит вечер», выезды, рейтинг, FAQ, контакты)
- `offer/privacy/consent/terms/rules.html` — документы (рыба, заполнить)
- `robots.txt`, `sitemap.xml`

**Личный кабинет игрока**
- `account.html` — вход (Telegram-виджет / телефон+пароль), онбординг, дашборд
- `app.html` — то же внутри Telegram Mini App (вход по initData)

**Админка и турнир-директор**
- `admin.html` — управление турнирами, записи, дубли по телефону
- `td.html` — консоль ведения турнира (чек-ин, рассадка, часы, закупы, финиш)
- `live.html` — табло для ТВ (часы, блайнды, метрики)

**Бэкенд `api/` (PHP)**
```
config.example.php  — шаблон конфига (копировать в config.php)
lib.php             — ядро: БД, сессии, Telegram-подписи, ачивки, слияние,
                      структура блайндов (default_levels), часы (tournament_clock)
auth_telegram.php   — вход через Telegram Login Widget
auth_webapp.php     — вход через Telegram Mini App (initData)
login_phone.php / register_phone.php — вход/регистрация по телефону+паролю
link_telegram.php   — привязка/слияние Telegram к аккаунту
onboarding.php      — сохранение ФИО/ник/телефон
me.php / logout.php / config_public.php
tournaments.php     — список турниров для сайта
register.php        — запись игрока на турнир
my.php              — данные игрока (записи, место за столом, рейтинг, ачивки)
rating.php          — рейтинг (?period=month|season|all)
td.php              — TD API: чек-ин, закупы, вылеты, рассадка, часы, финиш+очки
admin.php           — CRUD турниров, записи, результаты, дубли/слияние
clock.php           — публичные данные табло (часы/блайнды/метрики)
cron_reminders.php  — автонапоминания о турнирах (по cron)
schema.sql          — вся схема БД + демо-турниры
.htaccess           — защита config от прямого доступа
```

**Фронтовые скрипты `assets/js/`**: `main.js` (лендинг+рейтинг), `account.js`
(ЛК/мини-апп), `admin.js`, `td.js`, `live.js`, `mask.js` (маска телефона).
Стили — `assets/css/style.css` (все бренд-токены в `:root`).

## 2. Что нужно перед стартом
- Хостинг с **PHP 8.x + MySQL** (подойдёт виртуальный хостинг reg.ru/любой с Apache+nginx).
- **Домен на HTTPS** (валидный сертификат Let's Encrypt — обязательно для входа Telegram).
- **Telegram-бот** (создать у @BotFather, получить токен и username).

---

## 3. Установка (по шагам)
1. **БД:** создать базу MySQL → в phpMyAdmin импортировать `api/schema.sql`.
2. **Конфиг:** скопировать `api/config.example.php` → `api/config.php`, заполнить:
   - `tg_bot_token`, `tg_bot_username` — из @BotFather;
   - `db_*` — доступы к базе;
   - `cron_key` — любая случайная строка (`openssl rand -hex 16`);
   - `admins` — массив Telegram ID администраторов (узнать у @userinfobot).
   `config.php` НЕ коммитить в git (он в `.gitignore`).
3. **Файлы:** залить всё (вкл. `api/`) в корень сайта (`public_html/`).
4. **SSL:** в панели хостинга выпустить Let's Encrypt для домена.
5. **HTTPS-редирект:** корневой `.htaccess` уже форсит https (правь домен при необходимости).
6. **Бот:**
   - @BotFather → `/setdomain` → ваш домен (для Login Widget на сайте);
   - @BotFather → Bot Settings → Menu Button → URL `https://ВАШ_ДОМЕН/app.html` (для Mini App).
7. **Cron** (автонапоминания), каждые 15 мин:
   ```
   */15 * * * * /usr/bin/php /путь/к/сайту/api/cron_reminders.php >/dev/null 2>&1
   ```
8. Открыть `https://ВАШ_ДОМЕН/` и `…/account.html` — проверить вход и запись.
9. Первый администратор: войти под собой → выставить его Telegram ID в `admins`
   (или один раз `UPDATE users SET is_admin=1 WHERE tg_id=...`).

## 4. Что поменять под новый бренд

| Что | Где |
|---|---|
| Название клуба, тексты, секции | `index.html` (и заголовки в остальных `*.html`) |
| Домен в мета/OG/canonical/sitemap/robots | `index.html` (og/canonical), `sitemap.xml`, `robots.txt` |
| Логотип | `assets/img/logo-mark.png` (знак) и `logo-full.png` (с надписью), `favicon.png` |
| **Цвета и шрифты** | `assets/css/style.css` → блок `:root` (`--accent`, `--bg`, шрифты) |
| Username бота в подсказках | `app.html` (текст «@…_bot»), `account.js` (сообщения) |
| Адрес/контакты/карта | секция контактов в `index.html` |
| Документы (оферта и пр.) | `offer/privacy/consent/terms/rules.html` — заполнить плейсхолдеры |

### Настройки логики (бэкенд)
| Параметр | Где |
|---|---|
| **Структура блайндов** (уровни, минуты, аддон) | `api/lib.php` → `default_levels()` |
| **Делитель очков** (банк = фишки ÷ N) | `api/td.php`, действие `finalize` (сейчас `/ 100`) |
| **Кривая распределения** очков по местам | `api/td.php` → `finalize`, массив `$curve` |
| **Размер стола** для рассадки | поле турнира `table_size` (по умолчанию 9) |
| **Окна напоминаний** (24ч / 2ч) | `api/cron_reminders.php` |
| Демо-рейтинг на витрине | `assets/js/main.js` → `RATING` (заменить на реальный из `rating.php` при желании) |

## 5. Кэш (важно)
nginx на shared-хостинге кэширует статику надолго. Поэтому ссылки на css/js
версионированы (`?v=NNN`). **После любой правки css/js — увеличь номер версии**
во всех `*.html` (одной заменой `?v=629` → `?v=630`), иначе у пользователей
останется старый файл. HTML отдаётся с `no-cache` (см. `.htaccess`).

---

## 6. Чек-лист безопасности
- [ ] `api/config.php` не в git и не отдаётся наружу (проверь: `/api/config.php` → 403).
- [ ] Ротировать токен бота и пароли БД/хостинга после первичной настройки.
- [ ] SSL валидный (не самоподписанный).
- [ ] `robots.txt` закрывает `/api/` и `/account.html`; у служебных страниц `noindex`.

## 7. Поток одного турнирного вечера (для оператора)
Создать турнир в админке → **▶ Вести** → чек-ин игроков (выдать бэйджи с номерами)
→ 🎲 Жеребьёвка → ⏱ загрузить структуру → ▶ Старт (табло `live.html` на ТВ)
→ по ходу: закуп/аддон/вылет по номеру бэйджа → 🏁 Финиш → очки в рейтинг
и уведомления в Telegram автоматически.
